# 攻击方法说明

本目录实现的是面向语义分割模型的对抗攻击。大体可以分成三类：

- 白盒攻击：直接利用当前模型对输入的梯度更新扰动。
- 通用迁移攻击：保留图像分类里常见的动量、输入变换、平移不变等技巧，再适配到分割损失。
- 分割专用迁移攻击：针对像素级预测、特征结构或可迁移性分支专门设计的目标函数。

当前仓库里的攻击接口默认只接收一个模型，因此：

- 白盒攻击中，生成扰动的模型和评测模型天然相同。
- “迁移攻击”在当前实现里主要表示“采用迁移攻击的更新规则或目标函数”。
- 如果要做严格的 `source -> target` 跨模型迁移评测，需要在上层脚本里显式分离源模型和目标模型。

## 1. FGSM

FGSM 是最基础的单步攻击。它先计算输入对损失的梯度，再沿着梯度符号方向一次性走到扰动边界：

`x_adv = x + epsilon * sign(grad)`

直观上，它把每个像素都往“最能增大损失”的方向推一小步。优点是快，缺点是只有一步，往往欠优化，在分割这种高维像素预测任务上通常不如多步攻击强。

## 2. BIM

BIM 可以看成“多步版 FGSM”，也常被称为 iterative FGSM。它把 FGSM 的一次大步拆成多次小步，每一步后都投影回 `Linf` 扰动球内。当前仓库里 `BIM` 本质上是 `PGD` 的别名写法。

## 3. PGD

PGD 是最经典的多步白盒攻击。它反复执行三件事：

- 计算当前扰动样本的梯度
- 沿梯度符号方向更新
- 把结果投影回以原图为中心、半径为 `epsilon` 的 `Linf` 球内

如果加随机初始化，PGD 通常比 FGSM 更强，因为它可以逐步修正方向，而不是只相信第一步的局部线性近似。

## 4. SegPGD

SegPGD 是针对分割任务设计的 PGD 变体。核心思想是：分割图上不同像素的“攻击难度”不同，不能把所有像素一视同仁。

它把像素分成两类：

- 当前仍然预测正确的像素
- 当前已经预测错误的像素

然后用一个随迭代变化的系数 `lambda_t` 给两类像素分配不同权重。对 untargeted 攻击来说，它会更强调“还没被打掉的正确像素”，从而避免损失被已经错误的像素主导。这也是它通常比普通 PGD 更适合分割评测的原因。

## 5. CosPGD

CosPGD 的核心不是改更新规则，而是改像素损失的加权方式。它先计算模型 softmax 输出和目标 one-hot 向量之间的余弦相似度，再用这个相似度去缩放每个像素的交叉熵。

直观理解是：

- 如果某个像素当前输出与真值方向还比较接近，就给它相应的权重
- 整个攻击不再只看交叉熵大小，还结合了“预测方向与标签方向的角度关系”

这种做法在像素级任务里通常比直接平均 CE 更细，很多时候会比普通 PGD 更强。

## 6. SEA

SEA 是一个顺序集成攻击，不是单一损失。它按顺序执行三种互补攻击：

- `Mask-Cross-Entropy-Balanced`
- `Mask-Cross-Entropy`
- `JS-Avg`

它的核心思想是：不同损失会打击模型的不同薄弱点，顺序串联后再从中选择更坏的结果，往往比单一 PGD 类损失更强。

三个组成部分的含义是：

- `Mask-Cross-Entropy-Balanced`：只对当前仍然预测正确的像素计算损失，同时做类别平衡，避免大类主导
- `Mask-Cross-Entropy`：同样只关注当前正确像素，但不做类别平衡
- `JS-Avg`：用 Jensen-Shannon 散度衡量输出分布和目标分布的差异

对分割模型来说，SEA 的意义在于同时兼顾“难打的正确像素”“类别不均衡”“分布层面的偏移”。

## 7. MI-FGSM

MI-FGSM 是在多步攻击里加入动量。每一步不是直接用当前梯度，而是维护一个梯度累积项：

- 先把当前梯度按平均绝对值做归一化
- 再与历史梯度做指数式累积

这样做的作用是减少每一步方向的抖动，让更新方向更稳定，也更容易形成可迁移的扰动。

## 8. NI-FGSM

NI-FGSM 在 MI-FGSM 的基础上再加了 Nesterov look-ahead。也就是说，它不是在当前位置算梯度，而是先沿历史动量“预走一步”，再在这个前瞻位置计算梯度。

直观上，这样可以更早感知前方的损失地形，从而得到更稳定、通常也更有迁移性的更新方向。

## 9. DI²-FGSM

DI²-FGSM 的关键是输入多样化。每一步计算梯度前，先对输入做随机 resize 和 pad，再把变换后的图像送入模型。

它的作用是防止攻击过度依赖某个固定的输入坐标布局，提升扰动对缩放、平移等变换的鲁棒性，因此往往更利于迁移。

## 10. TI-FGSM

TI-FGSM 的关键是让梯度具有平移不变性。它会在每一步更新前，用高斯核对梯度做卷积平滑。

这样得到的不是某一个像素点特别尖锐的梯度，而是更平滑、更具有空间一致性的梯度场。通常这类梯度更容易跨模型迁移，因为它没那么依赖目标模型的局部高频细节。

## 11. NI+DI+TI

这是一个组合攻击，把三种迁移增强策略叠加起来：

- `NI`：Nesterov 前瞻 + 动量
- `DI`：输入多样化
- `TI`：梯度平滑

它的出发点很直接：动量负责稳定方向，输入变换负责减轻对固定坐标的依赖，平移不变负责让梯度更平滑。三者结合后，通常会比单独某一个迁移技巧更强，但也不一定总是最优。

## 12. DAG

DAG 指 Dense Adversary Generation。它是更典型的分割专用攻击，因为它不是只看整图平均损失，而是专门针对“仍然预测正确的像素”去构造密集目标。

它通常会：

- 为每个像素生成一个替代目标类别
- 只在当前还预测正确的像素上施加攻击目标
- 让这些像素的错误目标 logit 超过真实类别 logit

这样做的重点在于“逐像素地清除正确预测区域”，因此对分割模型尤其有效。

## 13. TASS

TASS 是面向分割迁移攻击的方案。当前仓库里的实现遵循参考仓库里实际使用的 `es_NI_DI_TI` 组合思路，本质上可以理解成“分割场景下的 NI + DI + TI 组合迁移攻击”。

它的原理重点不是重新定义一种完全不同的白盒损失，而是把：

- 动量优化
- Nesterov 前瞻
- 输入多样化
- 平移不变梯度

一起用于分割损失，从而得到更容易迁移的扰动。

## 14. TranSegPGD

TranSegPGD 的核心是“两阶段分支加权”。

第一阶段关注攻击性：

- 把像素分成“当前仍然预测正确”和“当前已经预测错误”两类
- 给仍然正确的像素更高权重
- 尽快把整张图上的正确像素打掉

第二阶段关注可迁移性：

- 计算当前对抗输出和原始干净输出之间的 KL 散度
- 用 KL 大小衡量像素的“可迁移属性”
- 把像素再分成“高可迁移”和“低可迁移”两类
- 对高可迁移分支给更高权重

所以它不是简单追求把源模型打坏，而是显式地区分“难攻性”和“可迁移性”。

## 15. FSPGD

FSPGD 是特征相似性驱动的 PGD。它不只看最终 logits，而是看中间特征图的结构关系。

它的基本思路是：

- 先提取干净图像的中间特征
- 对特征做归一化，并构造特征之间的相似性/亲和矩阵
- 再优化对抗样本的中间特征，让它与干净特征在两个层面上发生偏离

两个层面分别是：

- 对角相似性：同一位置的特征相似度
- 组合相似性：不同位置之间的成对关系

换句话说，FSPGD 不是只想把分类结果推错，而是想破坏干净特征内部的结构一致性。对于强调空间结构的分割模型，这类攻击有自己的针对性。

## 总结

如果从“为什么它们会越来越强”这个角度看，可以把这些方法理解成三条主线：

- 更新更充分：`FGSM -> PGD -> SegPGD / CosPGD / SEA`
- 迁移性更强：`MI / NI / DI² / TI / NI+DI+TI`
- 更贴近分割结构：`DAG / TASS / TranSegPGD / FSPGD`

前一类主要提升白盒破坏能力，后一类更多在想办法让扰动跨模型、跨结构也依然有效。对于语义分割，后两类通常比“直接把图像分类攻击搬过来”更合理。
